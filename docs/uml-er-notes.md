# UML и ER: краткий конспект

## UML-диаграммы

UML (Unified Modeling Language) используется для описания структуры и поведения системы.

Основные типы UML-диаграмм:

- Диаграмма вариантов использования (Use Case): показывает акторов и пользовательские сценарии.
- Диаграмма классов (Class): отражает сущности, поля, методы и связи между классами.
- Диаграмма последовательности (Sequence): отображает обмен сообщениями во времени.
- Диаграмма состояний (State Machine): показывает жизненный цикл объекта и переходы.
- Диаграмма активности (Activity): моделирует бизнес-процесс и ветвления логики.
- Диаграмма компонентов (Component): показывает модули приложения и их зависимости.
- Диаграмма развертывания (Deployment): отражает размещение сервиса по узлам инфраструктуры.

Зачем нужны UML-диаграммы:

- согласование архитектуры в команде;
- упрощение анализа требований;
- поддержка документации проекта и защиты решений.

## ER-диаграммы

ER (Entity-Relationship) описывает структуру данных и связи между сущностями БД.

Базовые элементы ER:

- Entity (сущность): таблица предметной области (например, `User`, `Ride`, `Payment`);
- Attribute (атрибут): поля сущности;
- Relationship (связь): связи вида 1:1, 1:N, M:N;
- Primary Key / Foreign Key: первичные и внешние ключи.

Зачем нужна ER-диаграмма:

- проектирование схемы БД до реализации;
- проверка целостности и корректности связей;
- упрощение написания миграций и запросов.

## Как применить к этому проекту

- UML Class: `User`, `Car`, `Ride`, `Payment`, `UserSession`.
- UML Sequence: `login -> access token -> refresh token`.
- ER: связи `Ride -> User`, `Ride -> Car`, `Payment -> Ride`, `UserSession -> User`.
